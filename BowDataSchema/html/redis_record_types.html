Record Types

There are 4 record types in the Schema DB. They work in a hierarchical fashion...

  • A Topic defines categories of communications handled on a channel. A Topic contains zero to many Plans.
  • A Plan identifies the set of Services which are available for a Topic.
  • A Service is a set of Request/Reponse or Publish/Subscribe or other collection of messages available within a Plan. The format of each service message is defined by a Schema.
  • A Schema identifies the acceptable format of a particular message.

The Harvest DB contains message reponse packets using Schema formats appropriate to their Plan.

The Log and Monitor DBs contain packets using Schema formats appropriate to the log or monitoring message Plan.
