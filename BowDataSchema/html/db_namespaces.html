Database Namespaces

  • DB 1 "Schema"
    • Message schema metadata and record templates.
    • Message record templates. For example, topic templates.
    • Primitive record templates. For example, "audit" fields template.
    • Use Schema Editor buttons and tools to explore and edit "Schema" DB.
  • DB 2 "Harvest"
    • Message response packets.
    • Contents of message responses are written to "Harvest" DB by the messaging gateway.
    • Receivers pull content from Redis using a token = a Redis unique key.
  • DB 3 "Log"
    • Detailed messaging system log.
    • Forensic info is written to "Log" DB for the messaging system Wire Tap.
    • Use the Monitor buttons and tools to explore "Log" DB and set its expiration rules.
  • DB 4 "Monitor"
    • Analytical messaging system log and summarizer.
    • Info is written to "Monitor" DB for the messaging system Wire Tap.
    • Use the Monitor buttons and tools to explore "Monitor" DB and set its expiration rules.
