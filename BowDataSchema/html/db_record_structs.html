Record Structures

Records of the same type share a structure.

  • Key Format is identical for all records of a given type.
    • A Topic definition record key structure ==> Category:Name:Version
      • Example: Redis:NamespacesSchema:0.1.0
    • A Service definition record key structure ==> TopicName:MessageName:MessageAction:Version
      • Example 1: Ontology:GetOntologyFile:request:0.1.0
      • Example 2: Ontology:GetOntologyFile:response:0.1.0
  • Values Format defines what fields are passed on a request or a response.
  • Audit Format is the same for all records. Consists of two fields:
    • HashID: hash_value
    • LastModified: timestamp+sequence_number
