Record Types

Schema DB has 4 main record types plus Primitives.

  • Topic records define communciation categories on a channel. A Topic contains zero to many Plans.
  • Plan identifies set of Services available for a Topic.
  • Service is a set of Request/Reponse or Publish/Subscribe messages available within a Plan. The format of each service message is defined by a Schema.
  • Schema identifies the acceptable format of a particular message.
  • Primitives define fundamental data types used in the system.

 

The Harvest DB contains message reponse packets using Schema formats appropriate to a Plan used in the Application.

 

The Log and Monitor DBs contain packets using Schema formats appropriate to a log or monitoring message Plan.
